1. Mô hình tollbooth (Tollbooth Model)

Câu chuyện: Quay lại trạm thu phí. Ta có một hàng xe dài (bộ đệm vô hạn) chờ trước cửa 1. Hai cửa thu phí hoạt động nối tiếp: xe vào cửa 1 trước, rồi cửa 2. Giữa hai cửa không có chỗ đợi — nếu cửa 2 đang bận, xe ở cửa 1 phải chờ tại chỗ dù đã được phục vụ xong.

{% include components/box-start.html type="definition" title='Mô hình hàng đợi Tollbooth' %}

Hệ thống gồm:

  • Dòng đến: Khách hàng (xe) đến theo quá trình Poisson với tốc độ \(\lambda\).
  • Server 1: Thời gian phục vụ phân phối mũ với tốc độ \(\mu_1\).
  • Server 2: Thời gian phục vụ phân phối mũ với tốc độ \(\mu_2\).
  • Bộ đệm: Hàng đợi trước server 1 có dung lượng vô hạn. Giữa server 1 và 2 không có bộ đệm.
  • Blocking: Nếu server 2 đang bận khi server 1 hoàn thành, xe bị chặn tại server 1.

Đây là hàng đợi dạng \(M/M/1 \to \cdot/M/1\) với blocking.

{% include components/box-end.html %}

Ký hiệu chính:

  • \(\lambda\): tốc độ đến (arrival rate)
  • \(\mu_1\): tốc độ phục vụ của server 1
  • \(\mu_2\): tốc độ phục vụ của server 2
  • \(1/\lambda\): thời gian trung bình giữa hai lần đến
  • \(1/\mu_k\): thời gian phục vụ trung bình của server \(k\)

2. Chuỗi Markov liên tục (CTMC) (Continuous-Time Markov Chain)

Để phân tích hệ thống, ta mô hình hóa nó bằng một chuỗi Markov thời gian liên tục (CTMC).

{% include components/box-start.html type="definition" title='Quá trình Markov của hệ tollbooth' %}

Định nghĩa quá trình Markov hai chiều \(\{I^*(t), J(t)\}\) trên không gian trạng thái \(S\):

  • \(I^*(t)\): số khách hàng trong hệ thống tại thời điểm \(t\) (bao gồm cả đang được phục vụ).
  • \(J(t)\): trạng thái kết hợp của hai server tại thời điểm \(t\).

\(J(t)\) nhận giá trị trong \(\{0, 1, 2\}\) theo quy ước:

\(J(t)\)Server 1Server 2Mô tả
0Rảnh (Idle)Rảnh (Idle)Cả hai server đều rảnh
1Bận (Busy)Rảnh (Idle)Server 1 đang phục vụ, server 2 rảnh
2Rảnh/BậnBận (Busy)Server 2 đang phục vụ
{% include components/box-end.html %}

Không gian trạng thái đầy đủ:

\[ S = \{(0, 0)\} \cup \{(i, j) : i \geq 1,\; j \in \{1, 2\}\} \]

Trạng thái \((0, 0)\) nghĩa là hệ thống trống. Khi có ít nhất 1 khách (\(i \geq 1\)), ít nhất một server phải đang bận, nên \(j \in \{1, 2\}\).

{% include components/box-start.html type="review" title='Ôn tập: CTMC (từ Chuỗi Markov, Ch. 5)' %}

Chuỗi Markov thời gian liên tục là quá trình ngẫu nhiên \(\{X(t) : t \geq 0\}\) sao cho:

  • Thời gian ở mỗi trạng thái phân phối mũ (exponential).
  • Khi rời trạng thái, chuyển sang trạng thái mới theo xác suất cố định.
  • Tính chất Markov: tương lai chỉ phụ thuộc hiện tại, không phụ thuộc quá khứ.

Xem chi tiết: Chuỗi Markov, Chương 5.

{% include components/box-end.html %}

3. Quy tắc vận hành (Operating Rules)

Hệ thống tollbooth hoạt động theo 5 quy tắc sau:

  1. FIFO: Khách hàng được phục vụ theo thứ tự đến (First-In-First-Out).
  2. Nối tiếp: Mỗi khách phải qua server 1 trước, rồi server 2.
  3. Không có bộ đệm giữa: Giữa server 1 và server 2 không có chỗ đợi.
  4. Blocking: Nếu server 1 hoàn thành nhưng server 2 đang bận, khách bị chặn tại server 1 (server 1 không thể nhận khách mới).
  5. Chuyển tức thì: Khi server 2 trống và có khách chờ (hoặc bị chặn) tại server 1, khách chuyển sang server 2 ngay lập tức.
{% include components/box-start.html type="summary" title='Tóm tắt' %}

Quy tắc blocking là đặc trưng quan trọng nhất: nó tạo ra sự phụ thuộc giữa hai server, khiến phân tích phức tạp hơn nhiều so với hai hàng đợi độc lập.

{% include components/box-end.html %}

4. Sơ đồ tốc độ chuyển (Transition Rate Diagram)

Sơ đồ dưới đây mô tả các tốc độ chuyển giữa các trạng thái (tương ứng Fig. 1 trong bài báo). Mỗi mũi tên chỉ hướng chuyển, nhãn là tốc độ chuyển.

Tại mức 0 (hệ thống trống hoặc có 1 khách):

  • \((0,0) \xrightarrow{\lambda} (1,1)\): Khách đến khi hệ trống → vào server 1.
  • \((1,1) \xrightarrow{\mu_1} (1,2)\): Server 1 xong → khách chuyển sang server 2.
  • \((1,2) \xrightarrow{\mu_2} (0,0)\): Server 2 xong → khách rời hệ, hệ trống.

Tại mức \(i \geq 2\) (có \(i\) khách trong hệ):

  • \((i,1) \xrightarrow{\lambda} (i+1,1)\): Thêm khách đến, server 1 đang bận (server 2 rảnh).
  • \((i,1) \xrightarrow{\mu_1} (i,2)\): Server 1 xong → khách chuyển sang server 2.
  • \((i,2) \xrightarrow{\lambda} (i+1,2)\): Thêm khách đến, server 2 đang bận.
  • \((i,2) \xrightarrow{\mu_2} (i-1,1)\): Server 2 xong → khách rời hệ, khách tiếp theo vào server 1.

5. Ma trận tốc độ chuyển (Transition Rate Matrices)

Cấu trúc QBD cho phép ta viết ma trận tốc độ chuyển dưới dạng block-tridiagonal. Các block ma trận chính (kích thước \(2 \times 2\), ứng với pha \(j \in \{1, 2\}\)):

{% include components/box-start.html type="formula" title='Ma trận tốc độ chuyển A₀, A₁, A₂' %}

Ma trận chuyển lên (thêm khách đến):

\[ A_0 = \begin{bmatrix} \lambda & 0 \\ 0 & \lambda \end{bmatrix} \]

Ma trận đường chéo (ở lại cùng mức):

\[ A_1 = \begin{bmatrix} -(\lambda + \mu_1) & \mu_1 \\ 0 & -(\lambda + \mu_2) \end{bmatrix} \]

Ma trận chuyển xuống (khách rời hệ):

\[ A_2 = \begin{bmatrix} 0 & 0 \\ \mu_2 & 0 \end{bmatrix} \] {% include components/box-end.html %}

Ma trận biên tại mức 0–1 (chuyển từ/đến trạng thái \((0,0)\)):

{% include components/box-start.html type="formula" title='Ma trận biên A₃, B₀, B₁' %} \[ A_3 = \begin{bmatrix} 0 \\ \mu_2 \end{bmatrix}, \quad B_0 = \begin{bmatrix} \lambda & 0 \end{bmatrix}, \quad B_1 = [-\lambda] \]

Trong đó \(A_3\) là vector cột (chuyển từ mức 1 về mức 0), \(B_0\) là vector hàng (chuyển từ mức 0 lên mức 1), và \(B_1 = [-\lambda]\) là tốc độ rời trạng thái \((0,0)\).

{% include components/box-end.html %}

6. Phương trình cân bằng (Balance Equations)

Tại trạng thái dừng, tốc độ vào = tốc độ ra cho mỗi trạng thái. Gọi \(\pi_{i,j} = \lim_{t \to \infty} \PP(I^*(t) = i, J(t) = j)\) là xác suất trạng thái dừng.

{% include components/box-start.html type="formula" title='Phương trình cân bằng (eqs 1–10 trong bài báo)' %}

Trạng thái \((0,0)\):

\[ \lambda \pi_{0,0} = \mu_2 \pi_{1,2} \tag{1} \]

Trạng thái \((1,1)\):

\[ (\lambda + \mu_1) \pi_{1,1} = \lambda \pi_{0,0} + \mu_2 \pi_{2,2} \tag{2} \]

Trạng thái \((1,2)\):

\[ (\lambda + \mu_2) \pi_{1,2} = \mu_1 \pi_{1,1} \tag{3} \]

Trạng thái \((i,1)\), \(i \geq 2\):

\[ (\lambda + \mu_1) \pi_{i,1} = \lambda \pi_{i-1,1} + \mu_2 \pi_{i+1,2} \tag{4} \]

Trạng thái \((i,2)\), \(i \geq 2\):

\[ (\lambda + \mu_2) \pi_{i,2} = \lambda \pi_{i-1,2} + \mu_1 \pi_{i,1} \tag{5} \] {% include components/box-end.html %}

Thêm điều kiện chuẩn hóa:

\[ \pi_{0,0} + \sum_{i=1}^{\infty} (\pi_{i,1} + \pi_{i,2}) = 1 \tag{6} \]

7. Điều kiện ổn định (Stability Condition)

{% include components/box-start.html type="theorem" title='Proposition 1 (Điều kiện ổn định)' %}

Hệ thống tollbooth ổn định (có phân phối dừng) khi và chỉ khi:

\[ \lambda < \frac{\mu_1 \mu_2}{\mu_1 + \mu_2} \]

Tương đương: \(\displaystyle\frac{1}{\lambda} > \frac{1}{\mu_1} + \frac{1}{\mu_2}\), nghĩa là thời gian trung bình giữa hai lần đến phải lớn hơn tổng thời gian phục vụ trung bình của cả hai server.

{% include components/box-end.html %}

Ý nghĩa trực quan: Tốc độ phục vụ hiệu dụng của hệ tandem là \(\mu_1 \mu_2 / (\mu_1 + \mu_2)\) — đây là nghịch đảo của tổng thời gian phục vụ trung bình. Nếu khách đến nhanh hơn tốc độ này, hàng đợi sẽ dài vô hạn.

{% include components/example-start.html title='Ví dụ: Kiểm tra điều kiện ổn định' %}

Cho \(\mu_1 = 2\), \(\mu_2 = 3\) (khách/phút). Tốc độ phục vụ hiệu dụng:

\[ \frac{\mu_1 \mu_2}{\mu_1 + \mu_2} = \frac{2 \times 3}{2 + 3} = \frac{6}{5} = 1.2 \text{ khách/phút} \]

Hệ ổn định khi \(\lambda < 1.2\). Nếu \(\lambda = 1\): ổn định. Nếu \(\lambda = 1.5\): không ổn định.

{% include components/example-end.html %}

8. Quá trình QBD & Spectral expansion (QBD Process & Spectral Expansion)

Cấu trúc lặp lại của phương trình cân bằng (4)–(5) cho thấy đây là quá trình QBD với mức (level) \(i\) và pha (phase) \(j \in \{1, 2\}\).

{% include components/box-start.html type="definition" title='Phương pháp Spectral Expansion' %}

Phương pháp spectral expansion tìm nghiệm dạng:

\[ \boldsymbol{\pi}_i = (\pi_{i,1},\; \pi_{i,2}) = \sum_{k} a_k x_k^i \boldsymbol{\psi}_k, \quad i \geq 1 \]

trong đó:

  • \(x_k\) là trị riêng (eigenvalue) của ma trận đa thức đặc trưng \(Q(x)\).
  • \(\boldsymbol{\psi}_k\) là vector riêng (eigenvector) tương ứng.
  • \(a_k\) là hệ số xác định bởi điều kiện biên.

Yêu cầu: \(|x_k| < 1\) để tổng hội tụ (xác suất hữu hạn).

{% include components/box-end.html %}

Ma trận đa thức đặc trưng:

{% include components/box-start.html type="formula" title='Ma trận đa thức đặc trưng Q(x)' %} \[ Q(x) = A_0 x^2 + A_1 x + A_2 = \begin{bmatrix} \lambda x^2 - (\lambda + \mu_1)x & \mu_1 x \\ \mu_2 & \lambda x^2 - (\lambda + \mu_2)x \end{bmatrix} \]

Trị riêng \(x\) được tìm từ điều kiện:

\[ \det[Q(x)] = 0 \]

Phương trình này là đa thức bậc 4 theo \(x\), và bài báo chỉ ra rằng nó có thể giải được bằng công thức tường minh. Chi tiết xem Chương 2: Lời giải dạng đóng.

{% include components/box-end.html %} {% include components/box-start.html type="summary" title='Tóm tắt chương' %}
  • Mô hình tollbooth = hàng đợi tandem với blocking, mô hình hóa bằng CTMC.
  • Không gian trạng thái: \(S = \{(0,0)\} \cup \{(i,j) : i \geq 1, j \in \{1,2\}\}\).
  • Ma trận tốc độ chuyển có cấu trúc QBD (block-tridiagonal lặp lại).
  • Điều kiện ổn định: \(\lambda < \mu_1\mu_2 / (\mu_1 + \mu_2)\).
  • Phương pháp spectral expansion: tìm trị riêng & vector riêng của \(Q(x)\).
{% include components/box-end.html %}
